# `0x52` Check Control Message Relay

IKE/IKI `0x80` â†’ CCM `0x30`

*Only applies to high cluster (IKE/IKI)!*

![Check Control Overview](../lcm/1a/cc.jpg)

Allows the cluster to write messages to the display while avoiding contention with check control. The CCM acts as arbiter.

#### Examples

    80 05 30 52 10 80 77
    80 19 30 52 11 94 20 20 20 52 41 4E 47 45 20 20 35 30 20 4B 4D 20 20 20 20 20 02

### Options

    # Byte 1
    
    # Priority? 0b0111_0000
    # TBC: does CCM display messages in order sent, or by this bitfield?
    RANGE               = 0b0001 << 4
    PROG                = 0b0010 << 4
    CODE                = 0b0011 << 4
    CODE_ALT            = 0b0100 << 4
       
    # Display 0b0000_0011
    # Same behaviour as 0x23 I believe
    CLEAR               = 0b00 # backlight off 
    SET                 = 0b01 # write and backlight on
    OVERWRITE           = 0b11 # 
    
    
    # Byte 2

    # TBC 0b1111_0000
    PROG                = 0b0100 << 4   # 0x4.
    CODE                = 0b0100 << 4   # 0x4.
    RANGE_CLEAR         = 0b1000 << 4   # 0x8.
    RANGE_IKI           = 0b1001 << 4   # 0x9. IKI
    RANGE_IKE           = 0b1010 << 4   # 0xa. IKE
     
    # Gong 0b0000_0111
    GONG_OFF            = 0b000
    GONG_HIGH_SINGLE    = 0b001
    GONG_HIGH_TONE      = 0b010
    GONG_HIGH_DOUBLE    = 0b011
    GONG_LOW_SINGLE     = 0b100

### String

All use cases appear write 20 characters.

- Range
- Prog.
- Code

## Use Cases

### Check Control: Range

    '   RANGE  99 KM     '

### OBC Remote Control ("PROG")

    '        PROG        '

The cluster display will read "PROG" while the OBC configuration remains active.

### Code ("CODE")

    '      CODE ----     '

The cluster display will display "CODE" until code is entered via BMBT.
